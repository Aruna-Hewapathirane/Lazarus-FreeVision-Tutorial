<b>Notes on the Code:</b>

Free Vision uses code page 437.
For this reason, umlauts should be used as character constants for error-free rendering.

//code+
ä = #132 Ä = #142
ö = #148 Ö = #153
ü = #129 Ü = #154
//code-

<b>General Note:</b>
Caution is advised when modifying text at runtime, e.g., <b>Label</b>, <b>StaticText</b>.

Since the text is stored as <b>PString</b>, it is important to ensure that sufficient memory is reserved for the text in the constructor (Init).

The easiest way to do this is as follows, thus leaving enough space for <b>world</b>.

//code+

StaticText := new(PStaticText, Init(Rect, 'Hello '));

StaticText^.Text^ := 'Hello world';

/code-

<b>TListBox</b>
Caution is advised when using the <b>TListBox</b> component. The <b>TList</b> assigned here must be cleaned up manually using a <b>Destructor</b>.

Examples can be found in the chapter <b>Lists and ListBoxes</b>.

<b>General Note 64-bit:</b>
When coding with 64-bit, errors can occur, <s>this primarily affects functions that use <b>FormatStr</b>.</s>
This is unfortunately quite noticeable in the row and column display of the windows.

<b>Turbo-Vision</b>

This tutorial contains elements that are <b>not</b> 100% compatible with <b>Turbo-Vision</b>.

Certain components and functions were only developed with Free Vision was introduced by FPC.

This includes, for example, TabSheet.

There are also features that are only available in Turbo Vision.
